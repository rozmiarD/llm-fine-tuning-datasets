# Accessibility Guidelines

Documentation in this repository should be readable and usable without depending on color, images, or visual layout alone.

## Use descriptive headings

Headings should describe the section content.

Good:

```text
## Dataset format
```

Weak:

```text
## Important
```

## Use meaningful links

Good:

```text
See [Dataset format](dataset-format.md).
```

Weak:

```text
Click here.
```

## Do not rely on badges alone

Badges are allowed, but the same information must also appear in text.

For example, if a badge says `CC BY 4.0`, the License section must also state the license.

## Keep examples copyable

Code and JSON examples should be complete enough to copy and test.

Avoid hidden assumptions when possible.

## Use plain language

Prefer short, direct sentences.

Avoid marketing language and vague claims.

## Avoid emoji as meaning

Emoji may be decorative, but must not be the only carrier of meaning.

This repository currently avoids emoji in documentation.
